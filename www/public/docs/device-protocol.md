# 快速开始

本文档介绍如何将设备接入 DraARL 平台。接入前请确保您已完成以下准备工作：

- 已完成注册并审核通过
- 已获取设备密码
- 拥有有效的业余电台呼号

---

## 设备接入流程

![设备接入流程](../../assets/device-access-flow.svg)

### 详细步骤

**步骤 1-2：注册与审核**

在平台完成注册，填写真实的业余电台呼号。管理员审核通过后，您的账号状态将变为"已激活"。

**步骤 3：获取设备密码**

登录后进入「个人中心」页面，您可以查看设备密码（脱敏显示）。如需查看完整密码，可点击显示按钮。您也可以自定义修改设备密码，长度为 6-10 位字母和数字组合。

**步骤 4：配置设备**

在您的设备客户端中填入以下认证信息：

- **用户名**：您的平台用户名
- **设备密码**：个人中心获取的设备准入密码
- **服务器地址**：平台提供的 UDP 服务器地址和端口

**步骤 5：开始通信**

设备连接成功后，会自动进行认证。认证通过后，您的呼号将由服务器自动填充。之后即可在同一群组内与其他设备进行语音和文本通信。

## 协议概述

**DraARLv1** (Dra Amateur Radio Link v1，麟链v1) 是 DraARL 平台的设备通信协议，用于设备与服务器之间的实时通信。

该协议专为业余无线电通信场景设计，采用 UDP 作为传输层协议，具有低延迟、高效率的特点。协议支持语音通联、文本消息、位置共享等功能，适用于各种客户端设备（移动端、桌面端、浏览器等）。

`UDP 传输` `大端序` `90字节固定头部` `变长负载` `Opus 16K 语音编码`

### 协议特点

- **轻量高效**：固定 90 字节头部，解析快速，适合嵌入式设备
- **安全认证**：设备密码 + 阶梯封禁机制，防止暴力破解
- **语音优化**：统一采用 Opus 16kHz 编码，音质与带宽平衡
- **群组通信**：支持多群组切换，同一群组内设备可互相通信
- **服务器互联**：支持多服务器部署，跨服务器语音转发

> 本协议为业余无线电通信设计，请遵守当地无线电管理法规。

## 报文结构

协议报文由固定头部（90字节）和变长数据区组成。所有多字节字段使用大端序（Big-Endian）。

```text
+--------+--------+--------+--------+--------+--------+--------+--------+
| 0-3    | 4-5    | 6-37   | 38-47  | 48     | 49     | 50     | 51-53  |
| Ver    | Length |Username|DevPass | Type   |DevModel| SSID   | DMRID  |
| 4B     | 2B     | 32B    | 10B    | 1B     | 1B     | 1B     | 3B     |
+--------+--------+--------+--------+--------+--------+--------+--------+
| 54-85  | 86-89  | 90+                                              |
|CallSign| Rsv    | DATA                                             |
| 32B    | 4B     | 变长                                             |
+--------+--------+--------+-----------------------------------------+

固定头部：90 字节
最小报文：90 字节
最大报文：800 字节（UDP 限制）
```

### 字段说明

| 偏移 | 长度 | 字段名 | 类型 | 说明 |
|------|------|--------|------|------|
| 0 | 4B | Version | string | 协议版本标识，固定为 "DraA" |
| 4 | 2B | Length | uint16 BE | 报文总长度（包含头部和数据） |
| 6 | 32B | Username | string | 用户名，UTF-8 编码，\0 填充 |
| 38 | 10B | DevicePassword | string | 设备准入密码，ASCII 字母数字 |
| 48 | 1B | Type | byte | 数据包类型 |
| 49 | 1B | DevModel | byte | 设备型号 |
| 50 | 1B | SSID | byte | 设备子号 (0-255) |
| 51 | 3B | DMRID | uint24 BE | DMR ID |
| 54 | 32B | CallSign | string | 业余电台呼号，服务器填充 |
| 86 | 4B | Reserved | - | 保留字段，填 0 |
| 90 | 变长 | DATA | []byte | 负载数据 |

## 数据包类型

协议定义了以下数据包类型，通过 Type 字段区分：

| 值 | 常量名 | 说明 | DATA 内容 |
|----|--------|------|-----------|
| 0 | TypeControl | 控制指令 | 控制命令数据 |
| 1 | - | 保留 | - |
| 2 | TypeHeartbeat | 心跳包 | 可选携带 GPS 位置信息 |
| 3 | TypeConfig | 设备配置 | 配置参数或查询响应 |
| 4 | TypeTextMessage | 文本消息 | UTF-8 编码文本 |
| 5 | TypeOpus16K | Opus 16K 语音 | Opus 16kHz 编码语音帧 |
| 6 | TypeServerVoice | 服务器互联语音 | 服务器间转发的语音数据 |
| 7 | TypeATPassThrough | AT 透传 | AT 命令透传 |

## Opus 语音格式

> **TypeOpus16K (Type=5)** 是平台的标准语音编码格式，采用 Opus 编解码器，专为业余无线电通信优化。

### 编码参数

| 参数 | 值 | 说明 |
|------|------|------|
| 采样率 | 16000 Hz | 16kHz 宽带语音 |
| 声道数 | 1 | 单声道 |
| 帧时长 | 60 ms | 每个 Opus 帧 60ms |
| 每帧采样数 | 960 | 16000 × 0.06 |
| 比特率 | 16-32 kbps | VOIP 模式 |

### WebSocket 客户端合并帧格式

为优化弱网性能，WebSocket 客户端采用 **2 帧合并发送** 策略：每 120ms 发送一个数据包，每个数据包包含 2 个 60ms 的 Opus 帧。

```text
+-------------+-------------+-------------+-------------+
| Frame1 Len  | Frame1 Data | Frame2 Len  | Frame2 Data |
| 2B          | N bytes     | 2B          | M bytes     |
+-------------+-------------+-------------+-------------+

Frame Len: 帧长度（大端序 uint16）
Frame Data: Opus 帧数据（60ms）
```

### 性能优化效果

| 项目 | 优化前 | 优化后 |
|------|--------|--------|
| 帧时长 | 20ms | 60ms |
| 发送间隔 | 20ms | 120ms |
| 每秒发包数 | 50 | ~8.3 |
| 单包大小 | 110-170B | 330-570B |
| 头部开销占比 | 50-80% | ~25-35% |
| 额外延迟 | - | +100ms |

> **解码兼容性**
>
> 接收端应同时支持：
> - **合并帧格式**：按长度前缀解析多个 Opus 帧
> - **单帧格式**：兼容无长度前缀的单帧数据（用于 UDP 设备直发）
>
> 判断逻辑：若首字节值 < 0x80，视为合并帧格式（长度前缀）；否则视为单帧格式。

## 安全限制

UDP 服务端实施以下安全策略，超出限制的数据包将被静默丢弃：

| 限制项 | 阈值 | 说明 |
|--------|------|------|
| 最大包体 | 800 字节 | 含 90 字节头部，防止异常大包攻击 |
| 包速率 | 25 包/秒 | 按 IP+Port 维度限速，防止 DDoS 攻击 |

> 正常语音通信（合并帧 ~8.3 包/秒，单帧 ~16.7 包/秒）远低于限速阈值，不会受到影响。

## 设备认证流程

设备通过发送心跳包（Type=2）进行认证。认证成功后，服务器会在响应中填充用户的呼号。

![认证流程时序图](../../assets/auth-sequence.svg)

> **安全机制**
>
> 为防止暴力破解，平台采用阶梯封禁机制：
> - 连续失败 3 次：封禁 10 秒
> - 再次失败：封禁 30 秒
> - 继续失败：封禁 60 秒、300 秒（递增）
>
> 封禁 Key：IP + Username

## 常见问题

### Q: 设备显示认证失败怎么办？

请检查：
1. 用户名是否正确
2. 设备密码是否正确（注意大小写）
3. 账号是否已审核通过
4. 是否因多次失败被封禁（等待封禁时间过期）

### Q: 设备多久没有数据会被判定离线？

服务器检测到设备 20 秒内无任何数据包（包括心跳和语音数据），将判定设备离线。建议设备端每 10-15 秒发送一次心跳包保持在线状态。

### Q: 支持哪些语音编码格式？

平台统一使用 Opus 16kHz 编码格式。这是目前业余无线电网络中音质与带宽平衡的最佳选择。

### Q: 如何切换通信群组？

登录平台后，在「设备管理」页面选择要操作的设备，点击「切换群组」按钮，选择目标群组后保存即可。设备会在下次心跳时获取新的群组配置。
